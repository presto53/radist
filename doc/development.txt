= Тестирование =

Для того, что бы убедиться в том, что изменения ничего не сломали есть набор unit и case тестов.

Сейчас юниттесты называются '''utest.py''', case тесты '''case_test.py'''.

== Покрытие тестами ==

Сейчас суммарно case+unit тестами покрывается более 80% исходников. Для некоторых модулей покрытие составляет > 98%

Покрытие хорошо проверять с помощью coverage.py ('''python-coverage''' пакет в ubuntu).

Для новой функциональности надо писать новые тесты. Желательно все, что можно проверять в первую очередь case--тестами.

= FreeBSD package =

Так как для установки используется стандартные freebsd пакеты, при добавлении/удалении файлов надо не забывать править 
'''pkg-plist'''. 

При изменении версии не надо забывать править '''setup.py''' и '''Makefile'''. 

= Предупреждение =

Да поразит заклятие импотенции того, кто будет править исходники, не с tabstop=4 и будет оставлять символы табуляции.

Еще неплохо было бы писать документацию для новых функций и обновлять старую.
